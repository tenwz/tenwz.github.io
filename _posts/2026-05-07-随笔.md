## 2026-05-07｜先做减法，再做加法

Apple Music 最初打动人的，不是它有什么，而是它**没有什么**。

没有评论区，没有热闹的社交功能，没有喧宾夺主的算法推荐。只有一个干净、纯粹的播放界面，让你和音乐之间没有任何阻隔。它先把自己做成了一只「简单的容器」，然后才在容器里一点点注入高音质、空间音频、精致到每一个细节的 UI 设计。

这个顺序很关键：**先做减法，赢得信任；再做加法，提升体验。**

我越来越觉得，做内容聚合平台也该是这个逻辑。

第一步不是急着堆功能、上推荐、做社交——而是给每个站点、每个 RSS 源一个**干净、纯粹、可以专注阅读的空间**。让用户先感受到「这里很安静，没有多余的东西」，他们才会愿意留下来。

然后在这个基础之上，再让每个人根据自己的个性去定制——字体、配色、排版、交互方式。不是平台强加一套统一的标准，而是提供足够的自由度，让每个用户都能拥有**属于自己的阅读体验**。

简单是起点，个性是终点。

## 2026-05-07｜偏差计量器

突然想到，可以做一个极简的趋势跟踪「偏差计算器」。

趋势跟踪的本质是反人性的——市场恐慌时你要进场，市场狂热时你要离场。但真到了那个时刻，没有人能靠意志力压住本能。恐惧和贪婪会把你从理性轨道上硬生生拽走。

所以与其考验人性，不如做一个**量化自己偏离理性程度的小工具**。

输入几个关键参数：当前价格、你的入场价、预设的止损/止盈位、以及你原定的仓位比例。它不光帮你算出盈亏，更重要的是算出一个**「情绪偏差指数」**——你现在的决策，和最初制定的计划偏离了多少。

比如：原本说好跌破 10% 就止损，你现在却在犹豫要不要「再等等看」。计算器会直接告诉你：「你当前的偏离值是 73%，建议执行原计划。」

**人不需要被说服，人只需要一个冰冷的数据，把自己从情绪里拉出来一秒钟。**

那一秒钟，就够了。

## 2026-05-07｜我在用 git 写博客，但用户不需要知道 git

刚才 push 的时候突然意识到一件事：我在写一篇关于「先做减法」的随笔，但我的发布流程却复杂得离谱——`git add`、`git commit`、`git push`。

普通用户需要这些吗？

**不需要。**

用户需要的是：选中文字，点发布，或者先存草稿箱。三个按钮，完事。

那我为什么在用 git？因为我是技术人，git 对我来说是肌肉记忆。但这就是问题所在——**我在用自己的惯性替用户做选择。**

如果我真的信奉「先做减法」，那发布流程应该是 invisible 的。用户写完，点一下，就上线了。后台是 git 还是数据库还是静态文件，用户凭什么要关心？

Apple Music 不会让用户知道它背后用了什么编码格式，什么缓存策略。用户只知道自己点了一下，音乐就响了。

**好的产品，把复杂留给自己，把简单交给用户。**

我在写「偏差计量器」的时候说「人不需要被说服，只需要一个冰冷的数据」。那我现在也给自己一个数据：当前发布流程的复杂度，是普通用户需要的 **∞ 倍**。

得改。
